# System Architecture of Soccer Highlight Generator

## 1. Executive Summary

The Soccer Highlight Generator is a sophisticated, multi-stage AI pipeline designed to automate the extraction of key moments from full-length soccer match videos. Instead of relying on metadata or external APIs, the system parses the raw visual data of the broadcast to identify actions (like goals, shots, and passes), classifies the camera perspective, and evaluates the contextual excitement of the clip to generate a broadcast-ready highlight reel.

The system is highly modular, featuring deep learning extraction networks, a deterministic rule-based scoring engine, and a scalable FastAPI backend that can be accessed via a Gradio web interface.

## 2. Core Architectural Pipeline

The end-to-end process involves five distinct stages:

### Stage 1: Feature Extraction (ResNet Backbone)
Raw video is massive and computationally expensive to analyze directly. The first step involves converting the video into dense mathematical representations.
- The pipeline samples the video at a fixed FPS (e.g., 2 FPS).
- Each frame is passed through a pre-trained **ResNet** convolutional neural network.
- The output is a high-dimensional feature vector for every frame, drastically reducing the data footprint while retaining critical visual information (player positions, field lines, ball location).

### Stage 2: Action Spotting Network (`ball_action_spotting`)
Identifying a "shot" or a "pass" requires temporal context—a single static image isn't enough.
- The feature vectors are fed into a temporal neural network (often an LSTM, TCN, or Temporal Convolutional Network).
- This network analyzes sequences of frames to predict the exact timestamp (timestamp spotting) of specific soccer actions.
- Supported classes usually include: `pass`, `cross`, `shot`, `goal`, `tackle`, `foul`.
- Output: A list of events with their respective timestamps and confidence scores.

### Stage 3: Camera View Segmentation (CALF)
Not all camera angles make for good highlights. A goal shown from a zoomed-in perspective of the manager's face is unwatchable as a highlight.
- The **CALF (Camera Angle Labeling Framework)** module analyzes the frames to classify the current broadcast view.
- Categories typically include: `Main Camera` (wide tactical view), `Close-up` (player face), `Replay`, `Crowd`.
- Output: A temporal timeline indicating which camera angle is active at any given second.

### Stage 4: Rule-Based Scoring Engine (`rules`)
Neural networks identify the *what* and *when*, but the rule engine determines the *value*.
- The engine ingests the action list and the camera timeline.
- It applies a custom heuristic algorithm to rank the clips.
  - **Action Density**: A clip containing a `pass` -> `cross` -> `shot` -> `goal` within 10 seconds receives a massive score multiplier.
  - **Camera Penalty**: Actions that occur entirely during a `Close-up` or `Replay` view are heavily penalized or discarded, ensuring only `Main Camera` live-play is highlighted.
- Output: A sorted JSON array of the top $K$ highlights with calculated start and end trim times.

### Stage 5: Video Trimming and Merging (`create_clips`)
The final rendering stage.
- Utilizing FFmpeg wrappers, the system slices the original high-resolution MP4 file according to the precise timestamps generated by the scoring engine (e.g., $T-5$ seconds to $T+3$ seconds around the action).
- If configured, it concatenates these individual clips into a single continuous highlight reel (`final_highlights.mp4`), applying basic crossfade transitions if necessary.

## 3. Application Architecture

The system is deployed using a modern microservice-style layout, enclosed within Docker.

### 3.1. Backend (FastAPI - `app_v2`)
Provides a robust REST API.
- Exposes endpoints like `POST /process` which accepts a video path and processing parameters.
- Manages the asynchronous execution of the heavy deep learning pipeline so HTTP requests do not time out.
- Returns JSON metadata about the generated clips.

### 3.2. Frontend (Gradio - `app_v1` / `app_v2`)
Provides a user-friendly web interface for non-technical users.
- Built with Gradio, it allows users to upload local video files.
- Provides UI sliders for adjusting the Top-K limit and selecting which action types to prioritize.
- Includes an embedded video player to preview the generated highlight reel immediately after processing.

## 4. Hardware and Performance Considerations

- **GPU Requirement**: The system is heavily reliant on PyTorch and CUDA. Running the ResNet extraction and Action Spotting networks on a CPU will result in processing times 10x to 50x longer than the duration of the match itself.
- **Disk I/O**: High-speed NVMe storage is highly recommended, as extracting features from a 4GB 90-minute 1080p MP4 requires rapid, sequential reading of massive amounts of binary data.
